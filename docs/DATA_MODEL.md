# Initial Data Model

This document defines the first version of the data model for Smart Expense AI.

The model is intentionally simple for the MVP, but designed to support future growth.

## Main Entities

## User

Represents an application user.

Suggested fields:

| Field | Type | Description |
| --- | --- | --- |
| id | UUID | Unique user identifier. |
| email | string | User email. |
| password_hash | string | Securely hashed password. |
| name | string | Optional display name. |
| plan | string | free, premium. |
| created_at | datetime | Creation date. |
| updated_at | datetime | Last update date. |

## Transaction

Represents a financial movement.

Suggested fields:

| Field | Type | Description |
| --- | --- | --- |
| id | UUID | Unique transaction identifier. |
| user_id | UUID | Owner user. |
| category_id | UUID | Assigned category. |
| merchant_id | UUID | Optional merchant. |
| amount | decimal | Transaction amount. |
| currency | string | Currency code, initially EUR. |
| transaction_type | string | expense or income. |
| description | string | Transaction description. |
| transaction_date | date | Date of the transaction. |
| is_recurring | boolean | Whether it is considered recurring. |
| source | string | manual, import, bank_api. |
| created_at | datetime | Creation date. |
| updated_at | datetime | Last update date. |

## Category

Represents a transaction category.

Suggested fields:

| Field | Type | Description |
| --- | --- | --- |
| id | UUID | Unique category identifier. |
| user_id | UUID | Owner user or null for system categories. |
| name | string | Category name. |
| type | string | expense or income. |
| color | string | Optional UI color. |
| icon | string | Optional UI icon. |
| is_system | boolean | Whether it is a default category. |
| created_at | datetime | Creation date. |

## Merchant

Represents a shop, company or service provider.

Suggested fields:

| Field | Type | Description |
| --- | --- | --- |
| id | UUID | Unique merchant identifier. |
| user_id | UUID | Owner user. |
| name | string | Merchant name. |
| normalized_name | string | Name prepared for matching. |
| created_at | datetime | Creation date. |

## Recurring Expense

Represents a detected recurring payment.

Suggested fields:

| Field | Type | Description |
| --- | --- | --- |
| id | UUID | Unique recurring expense identifier. |
| user_id | UUID | Owner user. |
| merchant_id | UUID | Related merchant. |
| category_id | UUID | Related category. |
| average_amount | decimal | Average payment amount. |
| frequency | string | monthly, weekly, yearly, unknown. |
| next_expected_date | date | Estimated next charge date. |
| confidence_score | decimal | Detection confidence from 0 to 1. |
| active | boolean | Whether the recurring expense is active. |
| created_at | datetime | Creation date. |
| updated_at | datetime | Last update date. |

## Alert

Represents an automatic warning generated by the system.

Suggested fields:

| Field | Type | Description |
| --- | --- | --- |
| id | UUID | Unique alert identifier. |
| user_id | UUID | Owner user. |
| alert_type | string | anomaly, duplicate_subscription, forecast_warning, price_increase. |
| severity | string | low, medium, high. |
| title | string | Short alert title. |
| message | string | Human-readable explanation. |
| related_transaction_id | UUID | Optional related transaction. |
| status | string | unread, read, dismissed. |
| created_at | datetime | Creation date. |

## Insight

Represents a generated financial observation.

Suggested fields:

| Field | Type | Description |
| --- | --- | --- |
| id | UUID | Unique insight identifier. |
| user_id | UUID | Owner user. |
| insight_type | string | spending_trend, saving_opportunity, category_growth, forecast. |
| title | string | Short insight title. |
| message | string | Human-readable explanation. |
| period_start | date | Analysis period start. |
| period_end | date | Analysis period end. |
| created_at | datetime | Creation date. |

## Relationships

```txt
User 1 ──── N Transaction
User 1 ──── N Category
User 1 ──── N Merchant
User 1 ──── N RecurringExpense
User 1 ──── N Alert
User 1 ──── N Insight

Category 1 ──── N Transaction
Merchant 1 ──── N Transaction
Merchant 1 ──── N RecurringExpense
```

## MVP Notes

For the first implementation, the most important tables are:

1. User.
2. Transaction.
3. Category.
4. Alert.

The rest can be added progressively as the intelligence layer grows.
